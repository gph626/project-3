# Feature Scope

## MVP Features

The first release should include only the functionality needed to prove the system works as a secure, searchable case management platform.

### Must Have

- secure login
- role-based access control
- dashboard with high-level metrics
- searchable case registry
- case detail page
- tasks and deadlines
- audit trail
- basic document metadata
- create, update, and close case flows

### Later Phase Features

- file uploads
- advanced reporting
- saved views
- notifications
- approval workflows
- retention rules
- exports

## In-Scope User Workflows

- create a new case
- search and filter cases
- inspect a case record
- update status and metadata
- assign and complete tasks
- review audit history
- manage basic document metadata

## Out Of Scope For MVP

- public self-service portal
- mobile-first native app
- advanced analytics platform
- multi-agency integrations
- automated legal reasoning or decision support

## Scope Guardrails

- keep the first release focused on one core workflow
- avoid building advanced document handling before the case record is stable
- keep the permission model simple enough to implement cleanly
- prioritize traceability over visual complexity
